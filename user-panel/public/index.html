上传项目镜像

📦
项目镜像 拖拽 .tar 文件到此处或点击选择
📋 不知道怎么打包镜像？点这里查看打包指南
⚠️ 重要：镜像必须是 linux/amd64 架构，否则部署后会启动失败。
🎁 新手福利：一键生成示例项目（复制粘贴即可）

执行以下命令，即可生成一个可部署的示例项目：

mkdir my-app && cd my-app

cat > server.js << 'SERVEREOF'
const http = require('http');
const PORT = process.env.PORT || 3000;
http.createServer((req, res) => {
  res.writeHead(200, { 'Content-Type': 'text/html; charset=utf-8' });
  res.end('<h1>部署成功！</h1>');
}).listen(PORT, () => console.log('服务启动: port=' + PORT));
SERVEREOF

cat > Dockerfile << 'DOCKEREOF'
FROM node:20-alpine
WORKDIR /app
COPY server.js .
ENV PORT=3000
EXPOSE 3000
CMD ["node", "server.js"]
DOCKEREOF

docker buildx build --platform linux/amd64 -t my-app:latest --load .
docker save my-app:latest -o my-app.tar

💡 执行完后会在当前目录生成 my-app.tar，直接上传即可。

1
准备 Dockerfile

确保你的应用监听 PORT 环境变量指定的端口（默认 3000）：

# Dockerfile 模板 - 确保你的应用监听 PORT 端口
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --omit=dev
COPY . .
ENV PORT=3000
EXPOSE 3000
CMD ["node", "server.js"]

💡 如果已有 Dockerfile，只需确保端口逻辑一致即可，无需完全照抄。

2
构建镜像（必须指定 linux/amd64） 
# macOS / Windows 用户必须加 --platform
docker buildx build --platform linux/amd64 -t my-app:latest --load .

# Linux 用户可以省略 --platform
docker build -t my-app:latest .
3
导出为 .tar 文件 
docker save my-app:latest -o my-app.tar

✅ 然后将生成的 my-app.tar 拖拽到上方上传区域即可。
